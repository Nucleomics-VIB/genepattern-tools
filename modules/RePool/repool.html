RePool

Summary

The RePool utility computes the sample volumes that you have to take to make a new pool that complements a previous one, in order to get a desired read distribution for both pools combined. It takes as input and writes as output an Excel file. Below you can download an Excel template. See documentation for more information about the Excel columns.

Description

See section input for how to format the input file.

RePool excludes samples that had already so many reads that they do not have to be rerun.

Input files

RePool takes as input an Excel file with the following columns :
  • SAMPLE : Samples names, one per row.
  • DESIRED.PERCENTAGE.RUN1.RUN2 : The final percentage of reads that you want to end up with for each sample, after combination of both pools. The values of this column should add to 100. If not, the values are rescaled by the script so that the sum adds to 100.
  • RUN1.VOLUME : The volume of each sample that was used to make the previous pool.
  • RUN1.BS.PERCENTAGE : The percentage of reads for each sample after sequencing the previous pool. Note that you have to calculate percentage by taking into account all lanes.
  • TOTAL.VOLUME.POOL1 : The total volume of the first pool.
  • pM.POOL1 : The amount of pM that was loaded when sequencing the first pool.
  • pM.POOL2 : The amount of pM that you intend to load when sequencing this new pool.
